# Contributing to the Federal Grant Compliance Framework

Thank you for your interest in contributing! This framework exists to help nonprofits and research institutes navigate federal grant compliance.

## Why Contribute?

- **Impact**: Help thousands of nonprofits protect their funding and serve communities
- **Learning**: Deepen your grants management expertise through practical application
- **Recognition**: All contributors are publicly acknowledged
- **Community**: Connect with practitioners, researchers, and compliance experts
- **Open Access**: Your work remains free and accessible forever

## Before You Start

1. **Review existing materials** to avoid duplication
2. **Check open issues** to see if someone is already working on similar improvements
3. **Read our [Code of Conduct](CODE_OF_CONDUCT.md)** - we maintain a respectful community
4. **Understand the license** - all contributions are licensed under CC BY 4.0

## Types of Contributions

### 1. Content Improvements
Fix errors, clarify explanations, add examples:
- Typos and grammatical errors
- Confusing explanations or procedures
- Missing examples or case studies
- Outdated regulatory citations
- Broken links

**How:** Small fixes can be submitted directly via pull request. For large changes, open an issue first to discuss.

### 2. Templates & Tools
Create or improve practical resources:
- Excel/Google Sheets calculators and trackers
- Checklists and decision trees
- Policy and procedure templates
- Workflow diagrams

**Requirements:**
- Must be editable (not password-protected)
- Include clear instructions for use
- Cite regulatory basis (e.g., "per 2 CFR 200.430")
- Test before submitting

### 3. Case Studies & Examples
Share implementation experiences:
- How you used the framework
- Challenges encountered and solutions
- Adaptations you made
- Lessons learned

**Format:** 2-5 pages including organization context, implementation approach, outcomes, and recommendations for others.

### 4. Translations
Help make the framework accessible:
- Spanish (high priority for U.S. nonprofits)
- Other languages as needed

**Requirements:** Native or fluent speaker who can maintain regulatory accuracy.

### 5. Testing & Feedback
Use the framework and report back:
- What worked well
- What was confusing
- What's missing
- Suggested improvements

**How:** Open an issue with the "feedback" label or email damoah.dao@gmail.com

## Contribution Process

### Small Changes (typos, minor edits)
1. Fork the repository
2. Make your changes
3. Submit a pull request
4. Wait for review (typically within 1-2 weeks)

### Large Contributions (new templates, major additions)
1. **Open an issue** first describing your planned contribution
2. **Discuss** with maintainers to ensure it fits the framework
3. **Get feedback** before investing significant time
4. **Develop** your contribution
5. **Submit** pull request with detailed description
6. **Respond** to review feedback
7. **Celebrate** when merged!

## Quality Standards

All contributions must meet these standards:

### Regulatory Accuracy
- ✅ Aligned with current 2 CFR 200 Uniform Guidance
- ✅ Specific regulatory citations provided (e.g., "2 CFR 200.403")
- ✅ Reviewed for technical accuracy
- ✅ Reflects current guidance (not outdated versions)

### Clarity
- ✅ Written in plain language (avoid jargon; define technical terms when necessary)
- ✅ Organized logically with clear headings
- ✅ Includes examples where helpful
- ✅ Appropriate for intended audience (grants managers, finance staff, executives, etc.)

### Usability
- ✅ Clear instructions provided
- ✅ Templates are editable and functional
- ✅ Tools have been tested
- ✅ Formatting is consistent with existing materials

### Accessibility
- ✅ PDFs are properly tagged for screen readers
- ✅ Images include descriptive alt text
- ✅ Color is not the only way to convey information
- ✅ Documents use clear heading structures

## Pull Request Guidelines

### Good PR Titles
- ✅ "Add cost allocation calculator for Domain 1"
- ✅ "Fix broken regulatory citation in procurement policy"
- ✅ "Clarify eligibility verification procedures in Domain 3"

### Poor PR Titles
- ❌ "Updates"
- ❌ "Fixed stuff"
- ❌ "Changes"

### PR Description Should Include
- **What changed** and why
- **Which domain(s)** it affects
- **Testing done** (if applicable)
- **Related issue** number (if applicable)

### Example Good PR Description
```
## Summary
Added budget variance analysis template for Domain 1 (Financial Management).

## Changes
- Created Excel template with variance calculation formulas
- Added instructions tab with step-by-step guidance
- Included sample data for testing
- Updated Domain 1 README to reference new template

## Testing
- Tested with sample grant data from three different grant sizes
- Verified formulas calculate correctly
- Confirmed compatibility with both Excel and Google Sheets

## Related Issue
Closes #15
```

## What We Cannot Accept

- Content that violates federal regulations or law
- Copyrighted materials without proper licensing and permission
- Commercial advertisements or promotions
- Inaccurate regulatory information
- Malicious code or security vulnerabilities
- Content that violates our Code of Conduct

## Review Process

1. **Automated checks** - License compatibility, formatting, broken links
2. **Peer review** - At least one community member reviews content
3. **Expert review** - Subject matter expert validates regulatory accuracy (for compliance content)
4. **Maintainer approval** - Final review and merge

**Timeline:** We aim for 1-2 week turnaround for most contributions.

By contributing, you agree that:
1. Your contributions will be licensed under CC BY 4.0

3. You understand contributions become part of a public resource

No formal signature required - your pull request indicates agreement.

## Reporting Issues

### Content Issues
- Inaccurate information
- Broken links
- Outdated regulatory citations
- Confusing explanations

### Technical Issues
- Broken templates or tools
- Formula errors
- Download problems
- Website issues

**How to Report:**
1. Check if issue already exists in [Issues](https://github.com/damoahdao-cyber/federal-grant-compliance-framework/issues)
2. If not, open new issue with appropriate label
3. Include:
   - Clear description of the problem
   - What you expected vs. what happened
   - Steps to reproduce (if applicable)
   - Screenshots if helpful
